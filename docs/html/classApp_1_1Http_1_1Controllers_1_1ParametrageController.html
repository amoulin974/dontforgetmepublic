Description détaillée

Controller for handling configuration and scheduling functionalities.

This controller manages various functionalities such as displaying the main configuration view, planning time slots (plages) for an enterprise, handling invitation responses, and performing CRUD operations on time slots.

◆ ajax()

ajax ( Request $request)

Handle AJAX requests for adding, updating, deleting, or modifying a time slot (plage).

This method processes different operations based on the "type" parameter in the request:

  • "add": Creates a new time slot with a default interval if none is provided.
  • "update": Updates the start time, end time, and date of an existing time slot.
  • "delete": Deletes an existing time slot after detaching its related activities.
  • "modify": Updates the interval of an existing time slot.
Paramètres
Request$requestThe HTTP request instance containing the operation type and data.
Renvoie
\Illuminate\Http\JsonResponse JSON response with the result of the operation.

◆ index()

index ( )

Display the main configuration view.

This method checks if the user is authenticated and is associated with any enterprise. If the user is not authenticated, it redirects to the login page. If the user does not work for any enterprise, it redirects to the home page. Otherwise, it returns the configuration view.

Renvoie
\Illuminate\Http\Response The response containing the configuration view or a redirect.

◆ indexPlage()

indexPlage ( Request $request,
Entreprise $entreprise )

Display the scheduling view for time slots (plages) for a given enterprise.

For AJAX requests, this method returns JSON data containing time slot details if the user is either an employee or an admin of the specified enterprise. For non-AJAX requests, it checks user authentication and association with the enterprise and then returns the appropriate view: a view for employees or a view for admins.

Paramètres
Request$requestThe HTTP request instance.
Entreprise$entrepriseThe enterprise for which to display time slots.
Renvoie
\Illuminate\Http\Response The response containing JSON data or the scheduling view.

◆ invit()

invit ( Request $request)

Process the response to an invitation.

This method handles invitation responses by updating or deleting the pivot record in the "travailler_entreprises" relationship for the authenticated user. Depending on the request type ('accept' or 'reject'), it updates the user's status or removes the association.

Paramètres
